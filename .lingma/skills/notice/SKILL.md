# 通用通知格式助手

本 Skill 用于帮助用户快速生成符合公司规范的通知文档，确保格式统一、专业。

## 核心功能

1. **自动添加部门前缀** - 在通知标题前自动添加"【XX部】"标识
2. **规范署名格式** - 部门署名不加粗，保持简洁
3. **通用模板设计** - 适用于任何部门和通知类型
4. **标准化结构** - 确保通知包含必要要素

## 使用流程

```
用户提出通知需求
  → 确认所属部门（如无则使用"XX部"）
  → 确定通知主题和内容要点
  → 生成标准化通知文档
  → 输出格式规范的通知文本
```

## 通知格式规范

### 基本结构

```
【部门名称】通知标题

正文内容...

                                    部门名称
                                    日期
```

### 详细规则

#### 1. 标题格式

- **有明确部门**：`【人力资源部】关于春节放假安排的通知`
- **无明确部门**：`【XX部】关于系统维护的通知`

#### 2. 正文格式

- 一级标题：可使用 `#`、`##` 或文字加下划线等方式
- 段落之间空一行
- 列表项清晰分明
- 语言正式、简洁

#### 3. 署名格式

- **部门署名不加粗**（重要！）
- 右对齐或居中对齐
- 日期格式：`YYYY年MM月DD日`

示例：
```
                                    人力资源部
                                    2026年04月27日
```

## 通用通知模板

```markdown
【{部门名称}】{通知标题}

{称呼}：

{开头说明通知目的和背景}

一、{主要内容第一部分}
{具体内容}

二、{主要内容第二部分}
{具体内容}

三、{主要内容第三部分}
{具体内容}

{结尾语，如"特此通知"、"请遵照执行"等}

                                    {部门名称}
                                    {日期}
```

## 使用指南

### 当用户提供通知需求时

1. **询问部门信息**
   - "请问这个通知是哪个部门发出的？"
   - 如用户未提供，默认使用"XX部"

2. **确认通知类型**
   - 会议通知 / 制度通知 / 活动通知 / 紧急通知 / 其他

3. **收集关键信息**
   - 通知主题
   - 时间、地点、人员等要素
   - 具体要求或说明
   - 截止日期或执行时间

4. **生成通知文档**
   - 按照标准格式输出
   - 确保部门署名不加粗
   - 标题前添加【部门】标识

### 常见场景处理

| 场景 | 处理方式 |
|------|----------|
| 用户说"帮我写个通知" | 询问部门和主题 |
| 用户说"行政部的会议通知" | 直接使用【行政部】前缀 |
| 用户未提部门 | 使用【XX部】并提醒用户补充 |
| 用户要求修改格式 | 根据用户需求调整，但保持署名不加粗 |
| 紧急通知 | 在标题中加入"紧急"字样 |

## 核心规范总结

✅ **必须遵守的规则：**

1. 标题前必须加【部门】标识
2. 部门署名永远不加粗
3. 日期使用中文格式（YYYY年MM月DD日）
4. 语言正式、简洁、清晰
5. 必要时添加"特此通知"作为结束语

❌ **避免的错误：**

1. 不要遗漏部门前缀
2. 不要将部门署名加粗
3. 不要使用口语化表达
4. 不要忘记署名和日期
