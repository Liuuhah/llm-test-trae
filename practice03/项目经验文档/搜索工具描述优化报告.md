# 搜索工具描述优化报告

## 🔍 问题描述

### 现象
AI 在接收到用户的搜索请求时，可能会尝试用 `list_directory` 或 `read_file` 工具去查找 log 文件，但不知道文件实际保存在 `D:\chat-log\log.txt`，导致：
1. AI 在项目目录下查找 → 找不到
2. AI 尝试其他常见位置 → 还是找不到
3. 最后才调用 `search_chat_history` 工具（如果它记得有这个工具的话）

**示例行为**：
```
用户: "帮我查一下第一条历史记录"

AI 的错误行为:
1. 思考：log 文件在哪里？可能在当前目录...
2. 调用 list_directory(".") → 查找项目目录
3. 结果：没找到 log.txt（因为文件在 D:\chat-log\）
4. 继续尝试其他位置...
5. 浪费时间和 Token
6. 可能最终还是调用了 search_chat_history，但效率很低
```

### 根本原因

LLM 的工具选择是基于**工具描述（description）**来决定的。原描述不够明确：

**原描述**：
```
搜索聊天历史记录。当用户想要查找之前的对话内容、回忆历史聊天信息、或询问之前讨论过的话题时，使用此工具。适用场景：1.用户输入以/search开头 2.用户表达'查找聊天历史'、'我之前说过什么'、'之前聊过什么'等意思 3.需要回忆之前的对话上下文。此工具会读取本地保存的聊天日志文件并返回相关记录。
```

**问题**：
- ❌ 没有明确告知文件路径
- ❌ 没有说明工具已封装好文件读取逻辑
- ❌ 没有禁止 AI 使用其他工具查找文件
- ❌ 缺少具体的使用示例

## 💡 为什么这样改有效？

### LLM 的工具选择机制

LLM 在选择工具时，主要依据：

1. **工具名称**（name）
2. **工具描述**（description）
3. **参数描述**（parameters.description）
4. **当前对话上下文**

其中，**工具描述**是最重要的决策依据。

---

### 优化策略

#### 1. 明确告知文件路径

```
**重要提示：聊天记录已自动保存在 D:\chat-log\log.txt 文件中**
```

**效果**：
- AI 知道文件在哪里
- 不需要自己去查找
- 避免无效的目录遍历

---

#### 2. 明确禁止无效操作

```
**请不要使用 list_directory 或 read_file 工具查找文件**
```

**效果**：
- AI 知道哪些工具不应该用
- 减少试错成本
- 提高准确性

---

#### 3. 说明工具已封装好

```
功能：读取 D:\chat-log\log.txt 文件，根据用户查询返回相关的历史记录
```

**效果**：
- AI 知道工具已经处理了文件读取
- 不需要自己处理编码、路径等问题
- 信任工具的封装

---

#### 4. 提供具体示例

```
使用示例：
- 用户问"查找聊天历史" → 调用 search_chat_history(query="查找聊天历史")
- 用户问"我之前说过篮球吗" → 调用 search_chat_history(query="篮球")
```

**效果**：
- AI 有明确的参考
- 学习正确的调用模式
- 提高泛化能力

## ⚠️ 注意事项

### 1. 工具描述的格式

**使用 Markdown 格式**：
- `**加粗**` 用于强调重点
- `\n\n` 用于分段
- 列表用于分条说明

**效果**：
- ✅ LLM 能理解 Markdown 格式
- ✅ 重点突出
- ✅ 结构清晰

---

### 2. 路径分隔符

**Windows 路径**：
```
D:\\chat-log\\log.txt
```

**注意**：
- JSON 字符串中需要使用双反斜杠 `\\`
- 实际路径是 `D:\chat-log\log.txt`

---

### 3. 示例的多样性

**提供的示例应该覆盖**：
- 通用查询（"查找历史"）
- 特定关键词（"篮球"）
- 具体问题（"第一条历史记录是什么"）

**效果**：
- ✅ AI 学习到不同的查询模式
- ✅ 提高泛化能力
- ✅ 适应各种用户需求

---

### 4. 与其他工具的协调

**确保其他工具的描述不会冲突**：

例如，`list_directory` 的描述应该是：
```
列出指定目录下的所有文件和子目录。适用于浏览文件系统、查找特定文件的位置等场景。
```

**不要说**：
```
可以用来查找 log 文件
```

**效果**：
- ✅ 工具职责清晰
- ✅ 避免混淆
- ✅ AI 正确选择
