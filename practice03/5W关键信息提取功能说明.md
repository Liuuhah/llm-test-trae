# 5W 关键信息自动提取功能说明

## 📋 功能概述

为 `practice03/chat_compress_client.py` 添加了智能关键信息提取功能，每 5 轮对话自动触发 LLM 按照 5W 规则（Who、What、When、Where、Why）提取关键信息，并保存到本地日志文件。

---

## 🎯 核心特性

### 1. 自动提取
- **触发条件**：每 5 轮对话自动触发
- **提取内容**：Who（谁）、What（什么）、When（何时）、Where（何地）、Why（为何）
- **保存位置**：`D:\chat-log\log.txt`
- **增量更新**：每次提取都追加到文件末尾，不覆盖历史数据

### 2. 灵活控制
- **单次跳过**：`skip_extract` - 跳过下一次提取
- **全局禁用**：`disable_extract` - 完全停止自动提取
- **重新启用**：`enable_extract` - 恢复自动提取
- **查看设置**：`extract_settings` - 显示当前配置

### 3. 智能提取
- **LLM 驱动**：使用 Qwen3.5 模型进行信息提取
- **精准提取**：只提取对话中明确提到的信息，不推测
- **格式规范**：严格按照 5W 格式输出
- **容错机制**：兼容 reasoning_content 和 content 字段

def __init__(self):

### 2. 提取检查逻辑

```python
def _check_and_extract_key_info(self):
    """检查是否需要提取关键信息，并在满足条件时执行"""
    
    # 检查是否全局禁用
    if not self.auto_extract_enabled:
        return
    
    # 检查是否单次跳过
    if self.skip_next_extract:
        self.skip_next_extract = False
        print("[关键信息提取] 用户跳过本次提取")
        return
    
    # 计算当前对话轮数
    rounds = len(self.chat_history) // 2
    
    # 每 extract_interval 轮对话触发一次
    if rounds > 0 and rounds % self.extract_interval == 0:
        print(f"\n[关键信息提取] 检测到第{rounds}轮对话，开始提取关键信息...")
        self._extract_5w_info()
```

### 3. 5W 提取提示词

```python
def _build_extract_prompt(self, recent_messages):
    """构建 5W 信息提取的提示词"""
    
    # 格式化对话历史
    conversation_text = ""
    for msg in recent_messages:
        role = "用户" if msg['role'] == 'user' else "AI助手"
        content = msg.get('content', '')
        if isinstance(content, list):
            content = str(content)
        conversation_text += f"{role}: {content}\n"
    
    prompt = f"""请分析以下对话内容，按照 5W 规则提取关键信息。

【对话内容】
{conversation_text}

【提取要求】
请严格按照以下格式输出，每个字段一行：
- Who: [参与者姓名或角色，如果没有明确提及则写"未提及"]
- What: [发生的主要事件或讨论的核心话题，用一句话概括]
- When: [具体时间或时间段，如果没有则写"未提及"]
- Where: [地点或场所，如果没有则写"未提及"]
- Why: [原因、目的或动机，如果没有则写"未提及"]

【注意事项】
1. 只提取对话中明确提到的信息，不要推测
2. 保持简洁，每个字段不超过 50 字
3. 如果某个信息在对话中出现多次，提取最重要的那个
4. 使用中文输出
5. 直接输出结果，不要有任何额外说明"""

    return prompt
```

### 4. LLM 请求发送

```python
def _send_extract_request(self, prompt, max_tokens=256, temperature=0.3):
    """发送提取请求到 LLM"""
    try:
        if self.base_url.startswith('https://'):
            conn = http.client.HTTPSConnection(self.host)
        else:
            conn = http.client.HTTPConnection(self.host)
        
        headers = {
            'Content-Type': 'application/json'
        }
        if self.token:
            headers['Authorization'] = f'Bearer {self.token}'
        
        data = {
            'model': self.model,
            'messages': [
                {'role': 'system', 'content': '你是一个专业的信息提取助手。'},
                {'role': 'user', 'content': prompt}
            ],
            'max_tokens': max_tokens,
            'temperature': temperature,
            'stream': False
        }
        
        conn.request('POST', f'{self.path}/chat/completions', json.dumps(data), headers)
        response = conn.getresponse()
        response_data = json.loads(response.read().decode())
        conn.close()
        
        return response_data
        
    except Exception as e:
        print(f"[提取请求错误] {str(e)}")
        return None
```

### 5. 响应解析

```python
def _parse_extraction_response(self, response_data):
    """解析 LLM 返回的提取结果"""
    
    if response_data is None:
        return None
    
    if 'choices' not in response_data or len(response_data['choices']) == 0:
        return None
    
    choice = response_data['choices'][0]
    message = choice.get('message', {})
    
    # 优先使用 reasoning_content（Qwen3.5）
    content = message.get('reasoning_content', '')
    if not content:
        content = message.get('content', '')
    
    if not content:
        delta = choice.get('delta', {})
        content = delta.get('content', '')
    
    if not content:
        return None
    
    content = content.strip()
    
    # 验证是否包含 5W 关键字段
    required_keywords = ['Who', 'What']
    has_required = any(keyword in content for keyword in required_keywords)
    
    if has_required:
        return content
    else:
        print("[解析警告] 返回内容不符合 5W 格式")
        return None
```

### 6. 日志保存

```python
def _save_to_log_file(self, extracted_info, round_number):
    """将提取的信息保存到日志文件（增量更新）"""
    
    log_dir = os.path.dirname(self.log_file_path)
    
    try:
        # 如果目录不存在，创建目录
        if not os.path.exists(log_dir):
            os.makedirs(log_dir)
            print(f"[日志保存] 创建目录: {log_dir}")
        
        # 获取当前时间戳
        timestamp = time.strftime("%Y-%m-%d %H:%M:%S")
        
        # 格式化日志条目
        log_entry = f"\n{'='*60}\n"
        log_entry += f"【记录时间】{timestamp}\n"
        log_entry += f"【对话轮次】第 {round_number} 轮\n"
        log_entry += f"{'='*60}\n"
        log_entry += extracted_info
        log_entry += f"\n{'='*60}\n"
        
        # 追加写入文件
        with open(self.log_file_path, 'a', encoding='utf-8') as f:
            f.write(log_entry)
        
        print(f"[日志保存] 成功保存到: {self.log_file_path}")
        return True
        
    except Exception as e:
        print(f"[日志保存] 错误: {str(e)}")
        return False
```

## ⚠️ 注意事项

### 1. 提取时机
- **触发点**：在 AI 回复完成后才触发
- **不影响对话**：提取过程不会阻塞正常对话流程
- **异步处理**：提取失败不会影响后续对话

### 2. 文件路径
- **默认路径**：`D:\chat-log\log.txt`
- **自动创建**：如果目录不存在会自动创建
- **降级方案**：如果 D 盘不可用，可以考虑修改为项目目录

### 3. 提取质量
- **只提取明确信息**：提示词强调不推测
- **格式验证**：检查是否包含 Who 和 What 字段
- **容错处理**：兼容 Qwen3.5 的 reasoning_content

### 4. 性能影响
- **轻量级请求**：max_tokens=256，temperature=0.3
- **快速响应**：通常 1-3 秒完成
- **低频触发**：每5轮才触发一次

### 5. 兼容性
- **与压缩功能独立**：两个功能互不影响
- **可同时启用**：压缩和提取可以同时工作
- **独立控制**：可以分别禁用其中一个
