# 跳过自动压缩功能使用说明

## 📋 功能概述

为 `practice03/chat_compress_client.py` 添加了灵活的压缩控制功能，用户可以：
- **单次跳过**：跳过下一次自动压缩
- **全局禁用**：完全禁用自动压缩功能
- **随时启用**：重新启用自动压缩
- **查看设置**：显示当前压缩配置状态

---

## 🎯 新增功能

### 1. 状态标志

在 `__init__` 方法中添加了两个新的属性：

```python
self.skip_next_compress = False      # 单次跳过标志（只对下一次有效）
self.auto_compress_enabled = True    # 全局自动压缩开关
```

### 2. 修改的逻辑

#### `_should_compress()` 方法
在执行原有的轮数和 token 检测之前，优先检查这两个标志：

```python
def _should_compress(self):
    """检查是否需要压缩聊天记录"""
    # 优先检查全局自动压缩开关
    if not self.auto_compress_enabled:
        return False
    
    # 检查单次跳过标志
    if self.skip_next_compress:
        self.skip_next_compress = False  # 重置标志
        print("\n[压缩跳过] 用户选择跳过本次压缩")
        return False
    
    # ... 原有的轮数和 token 检测逻辑
```

**工作流程：**
1. 如果 `auto_compress_enabled = False` → 直接返回不压缩
2. 如果 `skip_next_compress = True` → 重置为 False，返回不压缩
3. 否则执行原有的检测逻辑

## 💬 可用命令

### 1️⃣ skip_compress / 跳过压缩

**功能**：跳过下一次自动压缩（单次有效）

**使用示例**：
```
你: skip_compress
[设置] 下次将跳过自动压缩

你: （继续对话，即使超过5轮也不会触发压缩）
AI: （正常回复）

你: （再次对话，如果仍超过5轮，会正常触发压缩）
[压缩触发] 对话轮数(7)超过限制(5)
[压缩开始] 总共14条消息...
```

**适用场景**：
- 当前对话很重要，不想被压缩影响上下文
- 想再聊几轮后再让系统自动压缩
- 临时需要保留完整的对话历史

---

### 2️⃣ enable_compress / 启用压缩

**功能**：启用自动压缩功能（默认状态）

**使用示例**：
```
你: disable_compress
[设置] 已禁用自动压缩

你: （继续对话，不会触发压缩）
AI: （正常回复）

你: enable_compress
[设置] 已启用自动压缩

你: （继续对话，超过5轮时会正常触发压缩）
[压缩触发] 对话轮数(6)超过限制(5)
```

**注意**：启用时会自动清除 `skip_next_compress` 标志

---

### 3️⃣ disable_compress / 禁用压缩

**功能**：完全禁用自动压缩功能

**使用示例**：
```
你: disable_compress
[设置] 已禁用自动压缩

你: （继续对话，无论多少轮都不会触发压缩）
AI: （正常回复）

你: stats
[聊天统计]
  消息数量: 50 条
  对话轮数: 25 轮
  上下文长度: 15000 tokens
  压缩阈值: 5 轮 或 262144 tokens
```

**适用场景**：
- 长对话需要保留完整历史
- 调试时不想被压缩干扰
- 手动控制压缩时机

---

**功能**：显示当前压缩相关的所有设置

**使用示例**：

```
  enable_compress / 启用压缩 - 启用自动压缩

```

**显示内容**：
- 自动压缩开关状态

## 📝 注意事项

1. **skip_next_compress 是单次有效的**
   - 使用后会自动重置为 False
   - 只对下一次可能触发的压缩有效

2. **enable_compress 会清除 skip 标志**
   - 确保状态一致性
   - 避免混淆

3. **disable_compress 不影响已有压缩**
   - 只阻止未来的压缩
   - 已经压缩的历史不会被恢复

4. **命令不区分大小写**
   - `SKIP_COMPRESS`、`Skip_Compress`、`skip_compress` 都有效

5. **支持中英文命令**
   - `skip_compress` 和 `跳过压缩` 效果相同
   - 方便不同用户使用
