# Practice05: 动态技能加载系统 (Dynamic Skill Loading System)

## 1. 项目概述
本模块旨在为 LLM 客户端增加“动态技能管理”功能。通过扫描本地 `.agents/skills` 目录，自动发现并加载技能定义（SKILL.md），使 LLM 能够根据用户需求自主调用特定的技能指令集。

**核心目标：**
- 实现技能的自动发现与元数据提取。
- 建立技能内容与 LLM 上下文的动态注入机制。
- 验证技能调用的准确性与指令遵循能力。

## 3. 详细功能需求

### 3.1 技能扫描与解析 (YAML Front Matter)
- **路径规范**: 技能根目录为项目根目录下的 `.agents/skills`。
- **文件格式**: 每个技能必须是一个子目录，且包含 `SKILL.md` 文件。
- **解析逻辑**:
    -   读取 `SKILL.md` 的前 100 行。
    -   匹配 `---` 之间的内容。
    -   提取 `name` (字符串) 和 `description` (字符串)。
    -   **约束**: 严禁使用第三方库（如 `pyyaml`），必须使用 Python 标准库 `re` 进行解析。

### 3.2 上下文注入策略
- **初始状态**: 程序启动时，将技能列表以 JSON 格式放入 System Prompt。
- **动态加载**: 
    -   当 LLM 决定使用某技能时，会发起 `load_skill_content` 工具调用。
    -   系统将读取该技能的正文（Front matter 之后的部分）。
    -   将正文作为一条新的 System 消息插入到 `messages` 列表中，提示 LLM：“请严格遵照以下 [Skill Name] 的指令执行”。

### 3.3 安全性与健壮性
- **路径安全检查**: 在 `load_skill_content` 中，必须校验 `skill_name` 是否包含 `..` 或非法字符，防止路径遍历。
- **容错处理**: 如果 `SKILL.md` 不存在或格式错误，应返回友好的错误提示给 LLM，而不是抛出异常导致程序崩溃。

## 4. 接口定义 (API)

### 4.1 SkillManager 类
```python
class SkillManager:
    def __init__(self, skills_root_path: str):
        """初始化技能管理器"""
    
    def scan_and_cache(self) -> list[dict]:
        """扫描目录并缓存技能元数据"""
    
    def get_skill_json(self) -> str:
        """返回符合要求的 JSON 格式字符串"""
    
    def load_full_content(self, skill_name: str) -> str:
        """加载并返回技能的正文内容"""
```

## 5. 测试方案 (Test Plan)

### 5.1 测试环境准备
- 确保 `.agents/skills/notice/SKILL.md` 存在。
- **Notice Skill 要求**: 
    - 触发词：写通知、修改通知、润色通知。
    - 规则：必须以“XX部通知”开头；若用户未提供部门，默认使用“XX部”。

### 5.2 自动化测试用例 (`test_skills.py`)
1.  **用例 A：默认部门测试**
    -   **输入**: "帮我写一个五一劳动节放假的通知。"
    -   **预期行为**: 
        -   LLM 调用 `load_skill_content('notice')`。
        -   输出文本以 "**XX部通知**" 开头。
2.  **用例 B：指定部门测试**
    -   **输入**: "我是销售部的，请帮我写一个五一劳动节放假的通知。"
    -   **预期行为**: 
        -   LLM 调用 `load_skill_content('notice')`。
        -   输出文本以 "**销售部通知**" 开头。

### 5.3 验证指标
-   **工具调用成功率**: 100% 正确识别并调用技能。
-   **指令遵循度**: 输出格式完全符合 SKILL.md 中的约束。

## 6. 开发注意事项
1.  **编码问题**: 读取 `SKILL.md` 时务必指定 `encoding='utf-8'`。
2.  **JSON 格式化**: 发送给 LLM 的技能列表必须是合法的 JSON 字符串，建议使用 `json.dumps(..., ensure_ascii=False)`。
3.  **日志记录**: 在技能加载失败时，应在控制台打印详细的错误堆栈，方便调试。
4.  **性能优化**: 技能列表建议在内存中缓存，除非收到 `/reload_skills` 命令，否则不重复扫描磁盘。
