# AI 天气查询优化需求文档

## 📋 项目背景

我正在开发一个基于本地 LLM（Qwen3.5-9B）的聊天客户端，支持工具调用（Function Calling）。目前遇到了天气查询功能的严重问题，需要优化。

**技术栈：**
- 模型：Qwen3.5-9B (GGUF Q4_K_M 量化)
- 框架：LM Studio (本地部署)
- 语言：Python 3.x
- API：OpenAI 兼容接口

---

## 🔴 当前问题（需要修复）

### 问题 1：重复调用工具 7 次（严重）

**现象：**
用户查询"明天成都的天气怎么样？"，系统调用 `curl` 工具获取天气数据，但 AI 重复调用了 **7 次**相同的工具。

**日志证据：**
```
第 301 行：第 1 次请求
第 559 行：第 2 次请求
第 840 行：第 3 次请求
第 1129 行：第 4 次请求
第 1452 行：第 5 次请求
第 1765 行：第 6 次请求
第 2110 行：第 7 次请求
```

**影响：**
- 响应时间：69.68 秒（正常应该 10-15 秒）
- 浪费计算资源
- 用户体验极差

**期望行为：**
- 只调用 1 次工具
- 获取数据后直接返回给用户

---

### 问题 2：添加原始数据中不存在的信息

**现象：**
AI 在输出时添加了 tools.py 中没有提供的建议。

**AI 输出的内容（错误）：**
```markdown
## 💡 出行建议
- ✅ 早晨天气良好，适合户外活动
- 💨 傍晚有微风，体感凉爽
- 👕 温度温暖，建议穿短袖或轻薄衣物
- ☔ 全天无降水，无需带雨具  ← ❌ 这是 AI 自己加的
明天成都天气不错，是个适合出门的好日子！😊  ← ❌ 这也是 AI 加的
```

**实际 tools.py 提供的建议（正确）：**
```
💡 出行建议：
   ✅ 早晨天气良好
   👔 温度舒适，建议穿长袖T恤或薄外套
   💨 风力较小，体感舒适
```

**问题：**
- AI "幻觉"添加了不存在的信息
- 降低了准确性
- 可能引入错误建议

**期望行为：**
- 严格使用工具返回的内容
- 不要添加原始数据中没有的信息
- 不要做额外的推理或建议

---

### 问题 3：过度格式化

**现象：**
tools.py 已经格式化好的数据，AI 又做了一次格式化。

**AI 的处理：**
- `Clear` → `Clear（晴朗）`（添加中文翻译）
- 转换成 Markdown 表格
- 添加更多 Emoji
- 重新组织内容结构

**问题：**
- 浪费时间（增加推理时间）
- 可能引入翻译错误
- 增加了 token 消耗

**期望行为：**
- 直接使用工具返回的格式化文本
- 不要重新格式化
- 不要翻译或添加说明

---

### 问题 4：性能问题

**当前性能：**
- 响应时间：69.68 秒
- 工具调用次数：7 次
- Token 消耗：约 10,000+ tokens

**目标性能：**
- 响应时间：10-15 秒
- 工具调用次数：1 次
- Token 消耗：约 2,000-3,000 tokens

## 🎯 需求清单

### 需求 1：优化 System Prompt

**目标：** 明确告诉 AI 如何正确使用工具返回的数据

**要求：**
1. 添加工具使用规范
2. 禁止重复调用工具
3. 禁止添加原始数据中没有的信息
4. 禁止重新格式化数据

**期望的 System Prompt 结构：**
```
你是一个友好的AI助手，具有以下能力：
1. 文件操作：列出目录、读写文件、重命名、删除
2. 网页访问：使用 curl 工具访问网页
3. 知识库查询：当用户提到"文档仓库"、"文件仓库"、"知识库"、"查找文档"时，使用 anythingllm_query 工具查询本地文档

重要提示：
- 如果用户询问关于公司内部文档、政策、项目资料等问题，优先使用 anythingllm_query 工具
- 如果用户询问通用知识，直接回答即可
- 工具调用失败时，尝试其他方法或告知用户
- 从知识库返回结果时，只展示与问题直接相关的内容，过滤掉明显不相关的文档

【工具使用规范】（新增部分）
- 当工具返回数据后，直接使用该数据回答用户问题
- 不要重复调用相同的工具获取相同的数据
- 工具返回的内容已经过格式化，直接使用，不要重新格式化
- 不要添加原始数据中没有的信息或建议
- 不要翻译或解释工具返回的内容，直接展示即可
```

---

### 需求 2：优化 curl 工具描述

**目标：** 告诉 AI 天气数据已经格式化好，不要重复调用

**当前描述：**
```
通过HTTP请求访问网页，并返回网页内容。查询wttr.in天气时：
1. 必须使用英文城市名（如Chengdu、Beijing），不要使用中文
2. 必须添加 format=j1 参数获取JSON格式数据
3. 完整格式：https://wttr.in/{城市英文名}?format=j1
示例：https://wttr.in/Chengdu?format=j1
```

**期望的描述：**
```
通过HTTP请求访问网页，并返回网页内容。

【重要提示】
- 查询wttr.in天气时：
  1. 必须使用英文城市名（如Chengdu、Beijing），不要使用中文
  2. 必须添加 format=j1 参数获取JSON格式数据
  3. 完整格式：https://wttr.in/{城市英文名}?format=j1
  示例：https://wttr.in/Chengdu?format=j1
  
- 天气数据已经由工具格式化好，直接使用该数据回答用户
- 不要重复调用此工具获取相同城市的天气
- 如果需要查询其他城市或日期的天气，请明确说明

【禁止行为】
- 不要重复调用相同的URL
- 不要对返回的数据进行二次格式化
- 不要添加原始数据中没有的信息
```

## 📝 注意事项

1. **保持向后兼容**
   - 其他工具（文件操作、知识库查询）不受影响
   - 只优化天气查询相关的行为

2. **不要过度限制**
   - 允许 AI 在必要时调用其他工具

3. **保留灵活性**
